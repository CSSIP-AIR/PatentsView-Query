Query Tool FAQs

What is PatentsView?

The PatentsView application programming interface (API) provides web developers and researchers programmatic access to longitudinal data and metadata on patents, inventors, companies, and geographic locations.

PatentsView is a prototype patent data visualization and analysis platform intended to increase the value, utility, and transparency of US patent data. The initiative is supported by the Office of Chief Economist at the US Patent & Trademark Office (USPTO).

The PatentsView platform is built on a newly developed database that longitudinally links inventors, organizations, locations, and patenting activity since 1976. The data visualization tool, query tool, and flexible API enable a broad spectrum of users to examine the dynamics of inventor patenting activity over time and space. These tools also permit users to explore technology categories, assignees, citation patterns, and co-inventor networks.

PatentsView is intended to encourage the study and enhanced understanding of the intellectual property (IP) and innovation systems; to serve as a fundamental function of the government in creating "public good" platforms in these data; and to eliminate wasteful and redundant cleaning, converting, and matching efforts necessary for individual researchers to use these data, thus freeing up researchers to do what they do best-study IP, innovation, and technological change.

The current PatentsView API is a prototype, and the team welcomes feedback on data discrepancies.

What is the purpose of the PatentsView Query Tool?

The PatentsView Query Tool provides an easy to use web interface for querying data on patents, inventors, and assignees from the PatentsView database. Step 1 of Advanced Search defines the primary field of the output dataset: patents, inventors, or assignees. These primary fields correspond to the PatentsView API Endpoints.

The Quick Search is a simple query form that searches if any components of the query string separated by space can be found in patent title, patent abstract, inventor first and last name, assignee first and last name, or assignee organization. If all components of the string are found in at least one of the mentioned fields, the output dataset will contain the retrieved result from the PatentsView database.

Is there a way to preview results before submitting the full search?

Users can preview the results of their query in Step 4 before submitting the full search. The preview dataset contains up to top 20 rows of the full dataset and is presented in CSV, JSON, and XML format.

Where are the data field definitions that can be queried and retrieved using the PatentsView Query Tool?

Key terms in the patent data are defined in the Glossary.

The primary field defines the search groups available to define specific criteria. The Query Tool Data Dictionary provides definitions, search group, and category of all searchable fields.

The query returns no results but is seemingly valid. Why?

Data fields in the PatentsView database have particular format as parsed directly from the raw files published by the USPTO. The Query Tool validates the format of several fields and provides typeahead functionality for the CPC and USPC classes as well as Assignee Organization and all location fields. The Query Tool Data Dictionary also provides examples of valid query formats.
